# Level 2 -> 3

## 목표
파일명에 공백이 들어있는 파일을 읽기

## 사용한 명령어
```bash
ssh
ls
cat
```

### 실행 과정
ssh로 bandit 서버에 원격 연결 후에 디렉토리 내에 어떤 파일이 있는지를 확인
```bash
ls
```
`--spaces in this filename--`이 나옴
```bash
cat ./--spaces in this filename--
```
공백을 기준으로 세 개의 파일을 찾고서는 없다고 출력됨
```bash
cat "./--spaces in this filename--"
```
비밀번호가 출력됨

### 정리
파일명 내에 공백이 있을 경우에는 큰따옴표를 씌워주어야 정상적인 인식이 가능하다. 

그렇지 않으면 공백을 기준으로 여러 개의 파일명을 찾는 것으로 간주한다.

위 문제에서는 파일명이 특수기호로 시작하고 파일명에 공백도 들어있으니 `./`도 써주고 `" "`도 써주어야 정상적인 작동이 가능하다.

`" "` 대신에 공백 앞에 `\`를 추가해주는 것도 방법이다.
```bash
cat ./--spaces\ in\ this\ filename--
```
