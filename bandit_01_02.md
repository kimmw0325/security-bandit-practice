# Level 1 -> 2

## 목표
파일명이 특수문자로 된 파일을 읽기

## 사용한 명령어
```bash
ssh
ls
cat ./
```

### 실행 과정
ssh로 bandit 서버에 원격 연결 후에 디렉토리 내에 어떤 파일이 있는지를 확인
```bash
ls
```
파일 이름이 `-` 인 파일이 있는 것을 확인하고 내용 읽으려 시도
```bash
cat -
```
안먹힘. `./`를 파일명 앞에 붙이면 파일을 정상적으로 읽을 수 있음
```bash
cat ./-
```
비밀번호가 출력됨

### 정리
파일 이름이 `-`로 되어있어서 
```bash
cat -
```
를 시도함

위와 같은 형태로 작성하게 되면 `-`를 파일이름이 아니라 옵션이나 특수의미로 처리되기 때문에, 내가 원하는 기능을 하지 못한다.

```bash
cat ./-
```
`./`를 붙여주게 되면 명시적으로 현재 디렉토리 내에 있는 이름이 `-`인 파일을 읽으라는 뜻으로 정확히 인식하기 때문에 정상적인 작동이 가능

```bash
cat -- -
```
도 가능. `--` 뒤부터는 옵션이 아니라 인자로 해석하라는 뜻이다.

특수한 파일명을 다룰 때 명령어가 인자를 옵션이나 표준 입력으로 해석할 수 있다는 점을 확인했고, `./` 경로나 `--`를 사용하면 파일명을 명확히 지정 가능하다.
